Hello World - İlk C# Programımız 👋

Her programlama dilinde olduğu gibi, C#'a "Hello World" yazarak başlıyoruz. Bu basit program, bir C# uygulamasının ana giriş noktası olan `Main` metodunu ve konsola çıktı veren `Console.WriteLine` metodunu tanıtır.

// Örnek 1: Temel Hello World programı (Modern C#)
Console.WriteLine("Hello, World!");

// Örnek 2: Geleneksel yapı
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Merhaba Dünya!");
    }
}

// Örnek 3: Farklı bir metottan çağırma
using System;

class Program
{
    static void Main()
    {
        Selamla();
    }

    static void Selamla()
    {
        Console.WriteLine("Program başladı.");
        Console.WriteLine("Hoş geldiniz!");
    }
}

// Örnek 4: Argüman alan Main metodu
using System;

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine($"Program başladı. Argüman sayısı: {args.Length}");
        if (args.Length > 0)
        {
            Console.WriteLine($"İlk argüman: {args[0]}");
        }
    }
}

// Örnek 5: Farklı satırlarda çıktı
Console.Write("Merhaba ");
Console.Write("Dünya");
Console.WriteLine("!"); // İmleci bir alt satıra atar

// Örnek 6: Kullanıcıdan girdi alma
Console.Write("Lütfen adınızı girin: ");
string ad = Console.ReadLine();
Console.WriteLine($"Hoş geldin, {ad}!");

Sıra Sende! 🧠

  • Konsola "Sınav başladı!" yazdıran bir program yazın.
  • `Main` metodu içinden çağrılan `SinavMesaji` adlı bir metot oluşturun ve bu metot "Başarılar dileriz!" mesajını yazdırsın.
  • `Main` metoduna argüman olarak "Ali" ismini vererek, konsola "Merhaba Ali!" yazdıran bir program yazın.
  • Konsola sırasıyla "C#", "Öğreniyorum" ve "!" kelimelerini farklı `Write` metotlarıyla, ama aynı satırda yazdırın.
  • Yukarıdaki örnekleri kullanarak, "Adım [Ad], Soyadım [Soyad]" şeklinde bir çıktı veren program yazın.

Çözümler ✅

// 2. Main metodu içinden çağrılan SinavMesaji metodu

static void SinavMesaji()
{
    Console.WriteLine("Başarılar dileriz!");
}

// 3. Argüman alan Main metodu
static void Main(string[] args)
{
    if (args.Length > 0)
    {
        Console.WriteLine($"Merhaba, {args[0]}!");
    }
}
// Konsoldan çalıştırma: dotnet run Ali

// 4. Aynı satırda çıktı

Console.Write("Öğreniyorum");
Console.Write("!");

// 5. Ad ve Soyad çıktısı
string ad = "Ahmet";
string soyad = "Yılmaz";
Console.WriteLine($"Adım {ad}, Soyadım {soyad}");

Değişkenler ve Temel Veri Tipleri 📦

C#'ta değişkenler, `tip adı degiskenAdi = deger;` şeklinde tanımlanır. `var` anahtar kelimesi, derleyicinin tipi otomatik olarak belirlemesini sağlar.

// Örnek 1: String değişkeni (Metin)
string isim = "Ahmet";
Console.WriteLine(isim);
// Örnek 2: Integer değişkeni (Tam sayı)
int yas = 30;
Console.WriteLine($"Yaşım: {yas}");
yas = 31; // Değerini değiştirebiliriz
Console.WriteLine($"Yeni yaşım: {yas}");
// Örnek 3: Double (Ondalıklı) ve bool (Mantıksal) tipleri
double pi = 3.14159;
bool isActive = true; // Sadece true veya false olabilir
Console.WriteLine($"Pi sayısı: {pi}, Aktif mi: {isActive}");
// Örnek 4: 'var' anahtar kelimesi ile tip çıkarımı
var mesaj = "C# Öğreniyorum"; // Derleyici tipin string olduğunu anlar
var sayi = 20; // Derleyici tipin int olduğunu anlar
Console.WriteLine(mesaj);
Console.WriteLine(sayi);
// Örnek 5: Sabit (değiştirilemez) değişken - const
const double GRAVITY = 9.81;
// GRAVITY = 10; // Hata verir: const değişkeni yeniden atanamaz
Console.WriteLine($"Yer çekimi ivmesi: {GRAVITY}");
// Örnek 6: Kullanıcıdan string girdi alıp integer'a çevirme
Console.Write("Yaşınızı giriniz: ");
string yasGirdisi = Console.ReadLine();
int kullaniciYas = Convert.ToInt32(yasGirdisi);
Console.WriteLine($"Seneye {kullaniciYas + 1} yaşında olacaksınız.");

Sıra Sende! ✍️

  • Değeri "Masaüstü Bilgisayar" olan bir `string` değişkeni oluşturun.
  • Adı `stok` olan ve değeri 50 olan bir `int` değişkeni oluşturun, sonra bu değeri 45 olarak değiştirip her iki değeri de yazdırın.
  • Bir `bool` değişkeni tanımlayın ve ismini `isLoggedIn` koyun, değerini `false` olarak atayın.
  • Bir `char` değişkeni oluşturun ve değerini 'E' olarak atayın.
  • `decimal` tipinde, `const` anahtar kelimesini kullanarak `KDV_ORANI` adında bir sabit tanımlayın ve değerini 0.18 olarak belirleyin.
  • Kullanıcıdan en sevdiği rengi girmesini isteyin ve bunu bir `string` değişkende saklayıp ekrana yazdırın.

Çözümler ✅

// 1. string değişkeni
string urunAdi = "Masaüstü Bilgisayar";
Console.WriteLine(urunAdi);

// 2. int değişkeni
int stok = 50;
Console.WriteLine($"Stok: {stok}");
stok = 45;
Console.WriteLine($"Yeni stok: {stok}");

// 3. bool değişkeni
bool isLoggedIn = false;
Console.WriteLine($"Giriş yapıldı mı: {isLoggedIn}");

// 4. char değişkeni
char harf = 'E';
Console.WriteLine($"Karakter: {harf}");

// 5. decimal tipinde const değişkeni
const decimal KDV_ORANI = 0.18m;
Console.WriteLine($"KDV Oranı: {KDV_ORANI}");

// 6. Kullanıcıdan renk girdisi alma
Console.Write("En sevdiğiniz renk nedir? ");
string favoriRenk = Console.ReadLine();
Console.WriteLine($"Favori renginiz: {favoriRenk}");

Operatörler ➕➖✖️

Operatörler, değerler üzerinde işlem yapmak için kullanılan sembollerdir.

Aritmetiksel Operatörler

// Örnek 1: Aritmetiksel operatörler
int a = 10;
int b = 5;
Console.WriteLine($"Toplama: {a + b}");     // 15
Console.WriteLine($"Çıkarma: {a - b}");     // 5
Console.WriteLine($"Çarpma: {a * b}");      // 50
Console.WriteLine($"Bölme: {(double)a / b}"); // 2.0
Console.WriteLine($"Mod (Kalan): {a % b}"); // 0

Atama ve Mantıksal Operatörler

// Örnek 2: Atama operatörleri
int x = 10;
x += 5; // x = x + 5  (x'in yeni değeri 15)
Console.WriteLine($"x yeni değeri: {x}");
x *= 2; // x = x * 2  (x'in son değeri 30)
Console.WriteLine($"x'in son değeri: {x}");
// Örnek 3: Karşılaştırma operatörleri
int c = 10;
int d = 20;
Console.WriteLine($"c == d? {c == d}"); // Eşit mi? -> false
Console.WriteLine($"c != d? {c != d}"); // Eşit değil mi? -> true
Console.WriteLine($"d > c? {d > c}");   // Büyük mü? -> true

// Örnek 4: Mantıksal operatörler
bool isTrue = true;
bool isFalse = false;
Console.WriteLine($"VE (&&): {isTrue && isFalse}"); // isTrue VE isFalse -> false
Console.WriteLine($"VEYA (||): {isTrue || isFalse}"); // isTrue VEYA isFalse -> true
Console.WriteLine($"DEĞİL (!): {!isTrue}");        // isTrue DEĞİL -> false

// Örnek 5: Artırma ve Azaltma
int e = 5;
e++; // e'yi 1 artır
Console.WriteLine($"e'nin yeni değeri: {e}"); // 6
e--; // e'yi 1 azalt
Console.WriteLine($"e'nin son değeri: {e}"); // 5

// Örnek 6: Kullanıcıdan alınan iki sayıyı toplama
Console.Write("Birinci sayıyı girin: ");
int sayi1 = Convert.ToInt32(Console.ReadLine());
Console.Write("İkinci sayıyı girin: ");
int sayi2 = Convert.ToInt32(Console.ReadLine());
Console.WriteLine($"{sayi1} + {sayi2} = {sayi1 + sayi2}");

Sıra Sende! ⚙️

  • `sayi1` ve `sayi2` değişkenlerini tanımlayın (değerleri siz belirleyin) ve `sayi1`'in `sayi2`'den küçük veya eşit olup olmadığını kontrol edin.
  • `krediSkoru` değişkeni 700'den büyükse VE `gelir` değişkeni 5000'den büyükse true döndüren bir mantıksal ifade yazın.
  • `sayi` değişkenini 5'e bölün ve kalanı bir başka değişkene atayarak yazdırın.
  • `toplam` adında bir `int` değişkeni oluşturun, başlangıç değeri 0 olsun. Bu değişkene sırasıyla 10, 5 ve 20 ekleyerek son değerini yazdırın.
  • `sayac` adında bir değişken oluşturun ve `++` operatörünü kullanarak değerini bir artırıp konsola yazdırın.
  • Kullanıcıdan iki sayı alın ve bu sayıların çarpımını ekrana yazdırın.

Çözümler ✅

// 1. Karşılaştırma
int sayi1 = 15;
int sayi2 = 20;
Console.WriteLine($"Sayı1, Sayı2'den küçük veya eşit mi? {sayi1 <= sayi2}");

// 2. Mantıksal ifade
int krediSkoru = 750;
double gelir = 6000;
bool uygun = krediSkoru > 700 && gelir > 5000;
Console.WriteLine($"Kredi uygun mu? {uygun}");

// 3. Mod operatörü
int sayi = 17;
int kalan = sayi % 5;
Console.WriteLine($"5'e bölümünden kalan: {kalan}");

// 4. Atama operatörleri
int toplam = 0;
toplam += 10;
toplam += 5;
toplam += 20;
Console.WriteLine($"Toplamın son değeri: {toplam}");

// 5. Artırma operatörü
int sayac = 0;
sayac++;
Console.WriteLine($"Sayanın yeni değeri: {sayac}");

// 6. Kullanıcıdan alınan iki sayının çarpımı
Console.Write("Birinci sayıyı giriniz: ");
int num1 = Convert.ToInt32(Console.ReadLine());
Console.Write("İkinci sayıyı giriniz: ");
int num2 = Convert.ToInt32(Console.ReadLine());
Console.WriteLine($"Çarpım: {num1 * num2}");

Karar İfadeleri (If-Else) 🤔

`if`, `else if` ve `else` ifadeleri, belirli koşullara bağlı olarak kodun farklı bölümlerini çalıştırmak için kullanılır.

// Örnek 1: Basit if-else
int yas = 17;
if (yas >= 18)
{
    Console.WriteLine("Ehliyet alabilirsiniz. ✅");
}
else
{
    Console.WriteLine("Ehliyet için yaşınız yetersiz. ❌");
}
// Örnek 2: if-else if-else zinciri
int not = 85;
if (not >= 90)
{
    Console.WriteLine("Notunuz A 🥇");
}
else if (not >= 80)
{
    Console.WriteLine("Notunuz B 🥈");
}
else if (not >= 70)
{
    Console.WriteLine("Notunuz C 🥉");
}
else
{
    Console.WriteLine("Notunuz D 😔");
}
// Örnek 3: String karşılaştırması
string kullaniciAdi = "admin";
if (kullaniciAdi == "admin")
{
    Console.WriteLine("Yönetici girişi başarılı. 👑");
}
else
{
    Console.WriteLine("Kullanıcı adı yanlış.");
}
// Örnek 4: Birden fazla koşul
int saat = 22;
bool isWeekend = true;
if (saat > 20 && isWeekend)
{
    Console.WriteLine("Film izleme zamanı. 🍿");
}
else
{
    Console.WriteLine("Ders çalışma zamanı. 📚");
}
// Örnek 5: Tek satırlık if-else
int puan = 65;
if (puan > 50) Console.WriteLine("Geçtiniz! 🎉"); else Console.WriteLine("Kaldınız. 🙁");
// Örnek 6: Kullanıcıdan yaş alıp oy kullanma durumu kontrolü
Console.Write("Lütfen yaşınızı girin: ");
int girilenYas = Convert.ToInt32(Console.ReadLine());
if (girilenYas >= 18)
{
    Console.WriteLine("Oy kullanabilirsiniz. 🗳️");
}
else
{
    Console.WriteLine("Oy kullanamazsınız.");
}

Sıra Sende! 🎯

  • `sayi` adında bir değişken tanımlayın. Eğer bu sayı 0'dan büyükse "Pozitif", 0'dan küçükse "Negatif" ve 0'a eşitse "Sıfır" yazdıran bir kod yazın.
  • `kullaniciSifre` değişkeni "12345"e eşitse "Giriş başarılı.", aksi takdirde "Hatalı şifre." yazdırın.
  • `sicaklik` değişkeni 15'in altındaysa "Hava soğuk 🥶", 25'in üzerindeyse "Hava sıcak 🔥" ve aralıktaysa "Hava ılık 😊" yazdırın.
  • `isLoggedin` değişkeni `true` ise VE `isAdmin` değişkeni `true` ise "Yönetici paneli" yazdırın, aksi takdirde "Normal kullanıcı paneli" yazdırın.
  • Bir öğrencinin notu 50'den büyük veya eşitse "Geçti", değilse "Kaldı" yazdıran basit bir if-else ifadesi yazın.
  • Kullanıcıdan bir sayı alın ve sayının tek mi çift mi olduğunu ekrana yazdırın.

Switch İfadesi 🔀

`switch` ifadesi, bir değişkenin değerini bir dizi `case` (durum) ile karşılaştırır. Modern C#'ta desen eşleştirme (pattern matching) ile çok daha güçlü hale gelmiştir.

// Örnek 1: Klasik switch ifadesi
int gun = 3;
switch (gun)
{
    case 1:
        Console.WriteLine("Pazartesi");
        break;
    case 2:
        Console.WriteLine("Salı");
        break;
    case 3:
        Console.WriteLine("Çarşamba");
        break;
    default:
        Console.WriteLine("Geçersiz gün");
        break;
}
// Örnek 2: Switch ifadesi (C# 8.0 ve sonrası)
int not = 4;
string notAciklamasi = not switch
{
    5 => "Çok İyi",
    4 => "İyi",
    3 => "Orta",
    _ => "Geçersiz" // default durumu
};
Console.WriteLine(notAciklamasi);
// Örnek 3: Desen eşleştirme ile tip kontrolü
object obj = "Merhaba";
switch (obj)
{
    case int i:
        Console.WriteLine($"Bu bir integer: {i}");
        break;
    case string s:
        Console.WriteLine($"Bu bir string: {s}");
        break;
    default:
        Console.WriteLine("Bilinmeyen tip");
        break;
}
// Örnek 4: 'when' anahtar kelimesi ile ek koşul
int yas = 15;
string durum = yas switch
{
    < 12 => "Çocuk",
    >= 12 when yas < 18 => "Genç",
    _ => "Yetişkin"
};
Console.WriteLine(durum);
// Örnek 5: Gruplandırılmış case'ler
char harf = 'a';
switch (harf)
{
    case 'a':
    case 'e':
    case 'i':
    case 'o':
    case 'u':
        Console.WriteLine("Sesli harf");
        break;
    default:
        Console.WriteLine("Sessiz harf");
        break;
}
// Örnek 6: Kullanıcı girdisine göre işlem seçme
Console.WriteLine("1- Toplama\n2- Çıkarma");
Console.Write("Seçiminiz: ");
string secim = Console.ReadLine();
switch (secim)
{
    case "1":
        Console.WriteLine("Toplama seçildi.");
        break;
    case "2":
        Console.WriteLine("Çıkarma seçildi.");
        break;
    default:
        Console.WriteLine("Geçersiz seçim.");
        break;
}

Sıra Sende! 🧩

  • `mevsim` değişkenini ("Kış", "İlkbahar" vb.) kontrol ederek o mevsime ait bir ay (ör. "Ocak") yazdırın.
  • `httpStatusCode` değişkenini (ör. 200, 404) kontrol ederek "OK", "Not Found" gibi durum mesajlarını döndüren bir `switch` ifadesi yazın.
  • Bir `object` değişkeninin tipini kontrol edin. Eğer `double` ise "Bu bir double", `bool` ise "Bu bir bool" yazdırın.
  • Bir `int puan` değişkeni için, 90'dan büyükse "A", 80-89 arasındaysa "B", 70-79 arasındaysa "C" ve diğer durumlarda "F" döndüren bir `switch` ifadesi yazın (`when` kullanarak).
  • Haftanın gününü temsil eden bir sayı (1-7) için, 1-5 arası "Hafta içi", 6-7 arası "Hafta sonu" yazdıran bir `switch` ifadesi oluşturun.
  • Kullanıcıdan 1-3 arasında bir sayı girmesini isteyin ve `switch` ile seçilen menü öğesini ("Menü 1 Seçildi" gibi) yazdırın.

Çözümler ✅

// 1. Mevsim kontrolü
string mevsim = "Yaz";
string ay = mevsim switch
{
    "Kış" => "Ocak",
    "İlkbahar" => "Nisan",
    "Yaz" => "Temmuz",
    "Sonbahar" => "Ekim",
    _ => "Bilinmiyor"
};
Console.WriteLine(ay);

// 2. HTTP durum kodu
int httpStatusCode = 404;
string durumMesaji = httpStatusCode switch
{
    200 => "OK",
    404 => "Not Found",
    500 => "Internal Server Error",
    _ => "Unknown Status"
};
Console.WriteLine(durumMesaji);

// 3. Tip kontrolü
object veri = true;
switch(veri)
{
    case double d: Console.WriteLine($"Bu bir double: {d}"); break;
    case bool b: Console.WriteLine($"Bu bir bool: {b}"); break;
}

// 4. Puan kontrolü
int puan = 88;
string harfNot = puan switch
{
    >= 90 => "A",
    >= 80 and < 90 => "B",
    >= 70 and < 80 => "C",
    _ => "F"
};
Console.WriteLine($"Harf notunuz: {harfNot}");

// 5. Hafta içi / Hafta sonu
int gunNo = 6;
string gunTipi = gunNo switch
{
    >= 1 and <= 5 => "Hafta içi",
    6 or 7 => "Hafta sonu",
    _ => "Geçersiz gün"
};
Console.WriteLine(gunTipi);

// 6. Kullanıcı girdisine göre menü seçimi
Console.Write("Menü seçin (1-3): ");
string secim = Console.ReadLine();
switch(secim)
{
    case "1": Console.WriteLine("Menü 1 Seçildi"); break;
    case "2": Console.WriteLine("Menü 2 Seçildi"); break;
    case "3": Console.WriteLine("Menü 3 Seçildi"); break;
    default: Console.WriteLine("Geçersiz seçim"); break;
}

Döngüler 🔁

Döngüler, bir kod bloğunu belirli sayıda veya bir koşul doğru olduğu sürece tekrarlamak için kullanılır.

// Örnek 1: For döngüsü
for (int i = 0; i < 5; i++)
{
    Console.WriteLine($"i'nin değeri: {i}");
}
// Örnek 2: Foreach döngüsü (koleksiyonlarda gezinme)
var meyveler = new List<string> { "🍎 Elma", "🍐 Armut", "🍒 Kiraz" };
foreach (var meyve in meyveler)
{
    Console.WriteLine(meyve);
}
// Örnek 3: While döngüsü
int sayac = 0;
while (sayac < 5)
{
    Console.WriteLine($"Sayac: {sayac}");
    sayac++;
}
// Örnek 4: Do-while döngüsü
int i = 0;
do
{
    Console.WriteLine($"Do-while ile i: {i}"); // Bu blok en az bir kere çalışır
    i++;
} while (i < 3);
// Örnek 5: Break ve Continue
for (int num = 1; num <= 10; num++)
{

Metotlar ⚙️

Metotlar, belirli bir işi yapan, tekrar tekrar çağrılabilen kod bloklarıdır. Bu, kod tekrarını önler ve programı daha modüler hale getirir.

// Örnek 1: İki sayıyı toplayan ve sonuç döndüren metot
int Topla(int a, int b)
{
    return a + b;
}
int sonuc = Topla(10, 20);
Console.WriteLine($"Toplam: {sonuc}");
// Örnek 2: Değer döndürmeyen (void) metot

{
    Console.WriteLine($"Merhaba, {isim}! 👋");
}
SelamVer("Mert");
// Örnek 3: Expression-bodied metot (tek satırlık)
int Carpma(int x, int y) => x * y;
Console.WriteLine($"Çarpım: {Carpma(5, 4)}");
// Örnek 4: Varsayılan parametreli metot
void MesajGoster(string mesaj, string baslik = "Bilgi")
{
    Console.WriteLine($"[{baslik}] {mesaj}");
}
MesajGoster("İşlem başarılı."); // Başlık parametresi verilmedi, varsayılan kullanılır.
MesajGoster("Kayıt bulunamadı.", "Uyarı");

// Örnek 5: out parametresi ile birden fazla değer döndürme
bool Bolme(int bolunen, int bolen, out double sonuc)
{
    if (bolen == 0)
    {
        sonuc = 0;
        return false;
    }
    sonuc = (double)bolunen / bolen;
    return true;
}

if (Bolme(10, 2, out double bolmeSonucu))
{
    Console.WriteLine($"Sonuç: {bolmeSonucu}");
}

Console.Write("Karesi alınacak sayıyı girin: ");

Metotları Aşırı Yükleme (Overloading) 🏋️

Aynı isme sahip fakat farklı parametre imzalarına (parametre sayısı veya türü) sahip birden fazla metot oluşturmaya metot aşırı yükleme denir.

// Örnek 1: Farklı sayıda parametre
public class Hesaplayici
{
    public int Topla(int a, int b) => a + b;
    public int Topla(int a, int b, int c) => a + b + c;
}
var h = new Hesaplayici();
Console.WriteLine(h.Topla(5, 10));       // 15
Console.WriteLine(h.Topla(5, 10, 15)); // 30

// Örnek 2: Farklı türde parametre
public class Yazdirici
{
    public void Yazdir(string metin) => Console.WriteLine($"Metin: {metin}");
    public void Yazdir(int sayi) => Console.WriteLine($"Sayı: {sayi}");
}
var y = new Yazdirici();
y.Yazdir("Merhaba");
y.Yazdir(123);

// Örnek 3: Farklı tür ve sayıda parametre
public class AlanHesaplayici
{
    public double Alan(double kenar) => kenar * kenar; // Kare
    public double Alan(double kisaKenar, double uzunKenar) => kisaKenar * uzunKenar; // Dikdörtgen
}

// Örnek 4: Parametre sırasının farklı olması
public class Islem
{
    public void BilgiGoster(string ad, int yas) => Console.WriteLine($"{ad}, {yas} yaşında.");
    public void BilgiGoster(int yas, string ad) => Console.WriteLine($"Yaş: {yas}, İsim: {ad}");
}
var i = new Islem();
i.BilgiGoster("Ali", 30);
i.BilgiGoster(25, "Veli");

// Örnek 5: Constructor Overloading (Yapıcı metot aşırı yükleme)
public class Urun
{
    public string Ad { get; set; }
    public double Fiyat { get; set; }

    public Urun(string ad) { Ad = ad; Fiyat = 0; }
    public Urun(string ad, double fiyat) { Ad = ad; Fiyat = fiyat; }
}

Constructor (Yapıcı Metot) 🧱

Yapıcı metot (constructor), bir sınıftan nesne oluşturulduğunda otomatik olarak çağrılan özel bir metottur. Genellikle başlangıç değerlerini atamak için kullanılır.

// Örnek 1: Varsayılan ve parametreli constructor
public class Personel
{
    public string Ad { get; set; }
    
    public Personel() { Ad = "İsimsiz"; } // Parametresiz
    public Personel(string ad) { Ad = ad; } // Parametreli
}
var p1 = new Personel();
var p2 = new Personel("Canan");
Console.WriteLine(p1.Ad); // İsimsiz
Console.WriteLine(p2.Ad); // Canan

// Örnek 2: Sadece parametreli constructor
public class EvcilHayvan
{
    public string Tur { get; }
    public EvcilHayvan(string tur)
    {
        Tur = tur;
        Console.WriteLine($"Yeni bir {Tur} nesnesi oluşturuldu.");
    }
}
var kedi = new EvcilHayvan("Kedi");

// Örnek 3: Birden fazla constructor (overloading)
public class Kitap
{
    public string Baslik { get; set; }
    public string Yazar { get; set; }
    
    public Kitap(string baslik)
    {
        Baslik = baslik;
        Yazar = "Bilinmiyor";
    }
    public Kitap(string baslik, string yazar)

        Yazar = yazar;
    }
}
var kitap1 = new Kitap("Sefiller");
var kitap2 = new Kitap("Nutuk", "M. K. Atatürk");

// Örnek 4: Constructor'dan başka constructor'ı çağırma (: this)
public class Dikdortgen
{
    public double En { get; }
    public double Boy { get; }

    public Dikdortgen(double en, double boy) { En = en; Boy = boy; }

}

// Örnek 5: Nesne başlatıcı (Object Initializer)

    public string Ad { get; set; }

// Constructor yerine nesne başlatıcı kullanmak

Erişim Belirleyiciler 🔑

Erişim belirleyiciler, sınıfların ve üyelerinin (metot, özellik) nerelerden erişilebileceğini kontrol eder.

// Örnek 1: public - Her yerden erişilebilir
public class Banka
{
    public string Ad { get; set; } // Her yerden erişilebilir
}

// Örnek 2: private - Sadece tanımlandığı sınıf içinden erişilebilir
public class Kasa
{
    private double _bakiye = 1000; // Sadece Kasa sınıfı içinden erişilebilir

    public void ParaCek(double miktar)
    {
        if (_bakiye >= miktar) _bakiye -= miktar;
    }
}

// Örnek 3: protected - Tanımlandığı sınıf ve o sınıftan türeyen sınıflardan erişilebilir
public class Canli
{
    protected int Yas { get; set; }
}
public class Insan : Canli
{
    public void YasAta() { this.Yas = 30; } // 'Yas' protected olduğu için erişilebilir
}

// Örnek 4: internal - Sadece aynı proje (assembly) içinden erişilebilir
internal class ProjeAyarlari
{
    public static string ApiKey = "12345";
}

// Örnek 5: protected internal - Aynı proje içinden veya farklı projelerdeki türemiş sınıflardan erişilebilir
public class TemelSinif
{
    protected internal string Mesaj = "Merhaba";
}

Kapsülleme (Encapsulation) 💊

Kapsülleme, bir nesnenin verilerini (alanlarını) gizlemek ve bu verilere sadece kendi metotları (genellikle public property'ler) aracılığıyla erişilmesini sağlamaktır. Bu, veri bütünlüğünü korur.

// Örnek 1: Private field ve public property
public class Ogrenci
{
    private string _ad; // private alan (veri)

    public string Ad // public özellik (veriye erişim noktası)
    {
        get { return _ad; }
        set { _ad = value; }
    }
}

// Örnek 2: get/set içinde mantık
public class BankaHesabi
{
    private double _bakiye;
    public double Bakiye
    {
        get { return _bakiye; }
        private set // Sadece sınıf içinden değiştirilebilir
        {
            if (value < 0) throw new Exception("Bakiye negatif olamaz!");
            _bakiye = value;
        }
    }
    public void ParaYatir(double miktar) { Bakiye += miktar; }
}

// Örnek 3: Otomatik özellikler (Auto-implemented properties)
// Derleyici arka planda private bir alan oluşturur.
public class Urun
{
    public string Ad { get; set; }
    public double Fiyat { get; private set; } // Fiyat dışarıdan değiştirilemez
    public Urun(double fiyat) { Fiyat = fiyat; }
}

// Örnek 4: Sadece-getir (get-only) özellik
public class Dikdortgen
{
    public double Genislik { get; set; }
    public double Yukseklik { get; set; }
    public double Alan => Genislik * Yukseklik; // Değeri her çağrıldığında hesaplanır
}

// Örnek 5: init ile değişmezlik
public class Ayarlar
{
    public string Tema { get; init; } // Sadece nesne oluşturulurken atanabilir
}
var ayar = new Ayarlar { Tema = "Karanlık" };
// ayar.Tema = "Açık"; // Hata!

// Örnek 6: Kullanıcı girdisini doğrulamalı özelliğe atama
var bankaMusterisi = new BankaHesabi();
Console.Write("Yatırılacak tutarı girin: ");
double tutar = Convert.ToDouble(Console.ReadLine());
bankaMusterisi.ParaYatir(tutar); // Kapsülleme ile Bakiye'ye erişim
Console.WriteLine($"Yeni bakiye: {bankaMusterisi.Bakiye}");

Kalıtım (Inheritance) 👨‍👩‍👧

Kalıtım, bir sınıfın başka bir sınıftan özellik ve metotları miras almasını sağlar. Alt sınıf (derived class), üst sınıftan (base class) türer. Bu, kod tekrarını azaltır ve hiyerarşik bir yapı kurar.

// Örnek 1: Temel kalıtım
public class Canli // Base class (Temel Sınıf)
{
    public void NefesAl() => Console.WriteLine("Nefes alınıyor...");
}
public class Insan : Canli // Derived class (Türemiş Sınıf)
{
    public void Konus() => Console.WriteLine("Konuşuyor...");
}
var insan = new Insan();
insan.NefesAl(); // Miras alınan metot
insan.Konus();

// Örnek 2: base constructor'ını çağırma
public class Arac
{
    public int TekerlekSayisi { get; }
    public Arac(int tekerlekSayisi) { TekerlekSayisi = tekerlekSayisi; }
}
public class Otomobil : Arac
{
    // base(4) ile üst sınıfın constructor'ını çağırıyoruz.
    public Otomobil() : base(4) { } 
}
var otomobil = new Otomobil();
Console.WriteLine($"Otomobilin tekerlek sayısı: {otomobil.TekerlekSayisi}");

// Örnek 3: Metot geçersiz kılma (override)
public class Hayvan
{
    public virtual void SesCikar() // override edilebilmesi için virtual olmalı
    {
        Console.WriteLine("Hayvan bir ses çıkarıyor.");
    }
}
public class Kopek : Hayvan
{
    public override void SesCikar() // metodu geçersiz kıldık
    {
        Console.WriteLine("Hav hav! 🐶");
    }
}
var kopek = new Kopek();
kopek.SesCikar();

// Örnek 4: Kurucu parametrelerini geçirme
public class Kisi
{
    public string Ad { get; }
    public Kisi(string ad) { Ad = ad; }
}
public class Calisan : Kisi
{
    public string Pozisyon { get; }
    public Calisan(string ad, string pozisyon) : base(ad) // 'ad' parametresini base'e gönder
    {
        Pozisyon = pozisyon;
    }
}
var calisan = new Calisan("Ayşe", "Mühendis");
Console.WriteLine($"{calisan.Ad} adlı çalışanın pozisyonu: {calisan.Pozisyon}");

// Örnek 5: 'sealed' ile kalıtımı engelleme
public sealed class String { /* ... */ }
// public class MyString : String {} // Hata! Sealed sınıftan miras alınamaz.

Çok Biçimlilik (Polymorphism) 🎭

Çok biçimlilik, bir nesnenin farklı durumlarda farklı davranabilmesi yeteneğidir. Genellikle kalıtım yoluyla, üst sınıf referansının alt sınıf nesnelerini tutmasıyla ve `virtual`/`override` metotlarla sağlanır.

// Örnek 1: Temel Polymorphism
public class Sekil
{
    public virtual void Ciz() => Console.WriteLine("Bir şekil çiziliyor.");
}
public class Daire : Sekil
{
    public override void Ciz() => Console.WriteLine("Bir daire çiziliyor. ⚪");
}
public class Kare : Sekil
{
    public override void Ciz() => Console.WriteLine("Bir kare çiziliyor. ⬜");
}

Sekil sekil1 = new Daire();
Sekil sekil2 = new Kare();
sekil1.Ciz(); // Daire.Ciz() çalışır
sekil2.Ciz(); // Kare.Ciz() çalışır

// Örnek 2: Liste ile polymorphism
var sekiller = new List<Sekil>
{
    new Sekil(),
    new Daire(),
    new Kare()
};
foreach (var s in sekiller)
{
    s.Ciz(); // Her nesne kendi Ciz() metodunu çağırır
}

// Örnek 3: Metot parametresi olarak polymorphism
public void SekilCizdir(Sekil s)
{
    s.Ciz();
}
SekilCizdir(new Daire());
SekilCizdir(new Kare());

// Örnek 4: Abstract sınıflarla polymorphism
public abstract class Hayvan { public abstract void SesCikar(); }
public class Kedi : Hayvan { public override void SesCikar() => Console.WriteLine("Miyav!"); }
public class Kopek : Hayvan { public override void SesCikar() => Console.WriteLine("Hav!"); }

Hayvan h1 = new Kedi();
h1.SesCikar();

// Örnek 5: Interface ile polymorphism
public interface IUcabilir { void Uc(); }
public class Kus : IUcabilir { public void Uc() => Console.WriteLine("Kuş uçuyor."); }
public class Ucak : IUcabilir { public void Uc() => Console.WriteLine("Uçak uçuyor."); }

IUcabilir ucanNesne = new Kus();
ucanNesne.Uc();

Sıra Sende! 🎭

  • `Calisan` adında `virtual MaasHesapla()` metodu olan bir sınıf oluşturun.
  • `Mudur` ve `Isci` sınıflarını `Calisan`'dan türetin ve `MaasHesapla()` metodunu `override` edin.
  • Bir `List<Calisan>` listesi oluşturup içine `Mudur` ve `Isci` nesneleri ekleyin.
  • Bir `foreach` döngüsüyle listedeki tüm çalışanların maaşlarını hesaplatın.
  • `IOdemeYontemi` adında `Ode(double tutar)` metodu olan bir arayüz ve bu arayüzü uygulayan `KrediKarti` ve `Nakit` sınıfları oluşturun.
  • Kullanıcıdan "kare" veya "daire" girmesini isteyin. Girdiye göre ilgili `Sekil` nesnesini oluşturup `Ciz()` metodunu çağırın.

Soyut (Abstract) Sınıflar 👻

Soyut sınıflar, `abstract` anahtar kelimesiyle tanımlanır ve doğrudan nesnesi oluşturulamaz (`new` ile örneklenemez). Alt sınıflar için bir "şablon" görevi görürler. Alt sınıfların uygulamak zorunda olduğu soyut metotlar ve özellikler içerebilirler.

// Örnek 1: Soyut sınıf ve metot
public abstract class Sekil
{
    public abstract double AlanHesapla(); // Alt sınıflar bunu uygulamak ZORUNDA
}
public class Kare : Sekil
{
    public double Kenar { get; set; }
    public override double AlanHesapla() => Kenar * Kenar;
}
// var s = new Sekil(); // Hata! Soyut sınıftan nesne oluşturulamaz.

// Örnek 2: Soyut ve normal metotlar bir arada
public abstract class Veritabani
{
    public abstract void Baglan();    // Soyut metot
    public abstract void Kapat();     // Soyut metot
    public void Logla() => Console.WriteLine("İşlem loglandı."); // Normal metot
}
public class SqlVeritabani : Veritabani
{
    public override void Baglan() => Console.WriteLine("SQL'e bağlanıldı.");
    public override void Kapat() => Console.WriteLine("SQL bağlantısı kapatıldı.");
}

// Örnek 3: Soyut özellik
public abstract class Kitap
{
    public abstract int SayfaSayisi { get; }
}
public class Roman : Kitap
{
    public override int SayfaSayisi { get; }
    public Roman(int sayfaSayisi) { SayfaSayisi = sayfaSayisi; }
}
var roman = new Roman(450);
Console.WriteLine($"Roman {roman.SayfaSayisi} sayfa.");

// Örnek 4: Soyut sınıf hiyerarşisi
public abstract class Canli { }
public abstract class Hayvan : Canli { }
public class Kedi : Hayvan { } // Kedi, somut (concrete) bir sınıftır

// Örnek 5: Constructor'lı soyut sınıf
public abstract class Varlik
{
    public string Ad { get; }
    public Varlik(string ad) { Ad = ad; } // Constructor, türemiş sınıflar tarafından çağrılır
}
public class Insan : Varlik
{
    public Insan(string ad) : base(ad) { }
}

Sıra Sende! 👻

  • `Bina` adında, `Adres` özelliğine sahip bir `abstract` sınıf oluşturun ve `BilgiYazdir()` adında soyut bir metot ekleyin.
  • `Ev` sınıfını `Bina` sınıfından türetin ve `BilgiYazdir()` metodunu `override` ederek evin adresini yazdırsın.
  • `SoyutHayvan` sınıfı oluşturun, `YemekYe()` adında soyut bir metot ve `Uyu()` adında normal bir metot tanımlayın.
  • `Kedi` sınıfı `SoyutHayvan`'dan miras alsın ve `YemekYe` metodunu `override` etsin.
  • `Hesaplayici` adında, `Topla()` ve `Cikar()` adında soyut metotlara sahip bir `abstract` sınıf oluşturun.
  • Kullanıcıdan bir karenin kenar uzunluğunu alın, bir `Kare` nesnesi oluşturun ve `AlanHesapla()` metoduyla alanını yazdırın.

Çözümler ✅

// 1 ve 2. Bina ve Ev sınıfları
public abstract class Bina
{
    public string Adres { get; }
    public Bina(string adres) { Adres = adres; }
    public abstract void BilgiYazdir();
}
public class Ev : Bina
{
    public Ev(string adres) : base(adres) { }
    public override void BilgiYazdir() => Console.WriteLine($"Ev adresi: {Adres}");
}

// 3 ve 4. SoyutHayvan ve Kedi sınıfları
public abstract class SoyutHayvan
{
    public abstract void YemekYe();
    public void Uyu() => Console.WriteLine("Hayvan uyuyor.");
}
public class Kedi : SoyutHayvan
{
    public override void YemekYe() => Console.WriteLine("Kedi mama yiyor.");
}

// 5. Hesaplayıcı sınıfı
public abstract class Hesaplayici
{
    public abstract int Topla(int a, int b);
    public abstract int Cikar(int a, int b);
}

// 6. Kullanıcıdan alınan veriyle abstract sınıf kullanımı
Console.Write("Karenin kenar uzunluğunu girin: ");
double kenar = Convert.ToDouble(Console.ReadLine());
Kare k = new Kare { Kenar = kenar };
Console.WriteLine($"Karenin alanı: {k.AlanHesapla()}");

Arayüzler (Interface) 🤝

`interface`, bir sınıfın sahip olması gereken metotları ve özellikleri tanımlayan bir "sözleşmedir". Sadece imzaları içerir, gövdeleri yoktur (C# 8.0+ ile varsayılan gövde eklenebilir). Bir sınıf birden fazla `interface`'i uygulayabilir.

// Örnek 1: Temel interface
public interface IHareketEdebilir
{
    void Ilerle();
}
public class Araba : IHareketEdebilir
{
    public void Ilerle() => Console.WriteLine("Araba hareket ediyor.");
}

// Örnek 2: Birden fazla interface
public interface IHizlanabilir { void Hizlan(int hiz); }
public interface IDurabilir { void Dur(); }

public class Bisiklet : IHizlanabilir, IDurabilir
{
    public void Hizlan(int hiz) => Console.WriteLine($"Bisiklet {hiz} km/s hıza ulaştı.");
    public void Dur() => Console.WriteLine("Bisiklet durdu.");
}

// Örnek 3: Interface'te varsayılan metot (C# 8.0+)
public interface ILogger
{
    void Log(string message);
    void LogError(string message) => Console.WriteLine($"HATA: {message}"); // Default implementation
}
public class ConsoleLogger : ILogger
{
    public void Log(string message) => Console.WriteLine(message);
}
var logger = new ConsoleLogger();
logger.Log("Bilgi mesajı.");
logger.LogError("Kritik hata!"); // Varsayılan metot çağrılabilir

// Örnek 4: Interface'te özellikler
public interface IVerimli
{
    int Verim { get; }
}
public class Ampul : IVerimli
{
    public int Verim { get; }
    public Ampul(int verim) { Verim = verim; }
}

// Örnek 5: Farklı sınıfların aynı interface'i uygulaması
public interface IYuruyebilir { void Yuru(); }
public class At : IYuruyebilir
{
    public void Yuru() => Console.WriteLine("At yürüyor. 🐎");
}
public class Insan : IYuruyebilir
{
    public void Yuru() => Console.WriteLine("İnsan yürüyor. 🚶");
}

Sıra Sende! 🤝

  • `IOynanabilir` adında, `Oyna()` ve `Dur()` metotlarına sahip bir interface tanımlayın.
  • `MuzikCalar` sınıfı `IOynanabilir` interface'ini uygulasın.
  • `IVeritabani` adında, `Ekle()` ve `Sil()` metotlarına sahip bir interface oluşturun.
  • `SqlServer` ve `Oracle` sınıfları `IVeritabani` interface'ini uygulasın.
  • `Ekran` sınıfı, `IAcilabilir` (Ac metodu) ve `IKapanabilir` (Kapa metodu) adında iki farklı interface'i uygulasın.
  • Kullanıcıdan "sql" veya "oracle" seçmesini isteyin ve seçime göre ilgili `IVeritabani` nesnesini oluşturup `Ekle()` metodunu çağırın.

Çözümler ✅

// 1 ve 2. Oynanabilir arayüzü ve MuzikCalar
public interface IOynanabilir { void Oyna(); void Dur(); }
public class MuzikCalar : IOynanabilir
{
    public void Oyna() => Console.WriteLine("Müzik çalıyor.");
    public void Dur() => Console.WriteLine("Müzik durdu.");
}

// 3 ve 4. IVeritabani ve sınıfları
public interface IVeritabani { void Ekle(); void Sil(); }
public class SqlServer : IVeritabani {
    public void Ekle() => Console.WriteLine("SQL Server'a eklendi.");
    public void Sil() => Console.WriteLine("SQL Server'dan silindi.");
}
public class Oracle : IVeritabani {
    public void Ekle() => Console.WriteLine("Oracle'a eklendi.");
    public void Sil() => Console.WriteLine("Oracle'dan silindi.");
}

// 5. Ekran sınıfı
public interface IAcilabilir { void Ac(); }
public interface IKapanabilir { void Kapa(); }
public class Ekran : IAcilabilir, IKapanabilir
{
    public void Ac() => Console.WriteLine("Ekran açıldı.");
    public void Kapa() => Console.WriteLine("Ekran kapandı.");
}

// 6. Kullanıcı seçimiyle arayüz kullanımı
IVeritabani db;
Console.Write("Veritabanı seçin (sql/oracle): ");
string secim = Console.ReadLine();
if (secim == "sql") {
    db = new SqlServer();
} else {
    db = new Oracle();
}
db.Ekle();

Static Sınıflar ve Üyeler 🏛️

`static` üyeler (metotlar, özellikler), bir sınıfın nesnesini oluşturmadan (`new` kullanmadan) doğrudan sınıf adıyla erişilebilen üyelerdir. `static` bir sınıf ise sadece `static` üyeler içerebilir ve nesnesi oluşturulamaz.

// Örnek 1: Static metot ve özellik
public class Matematik
{
    public static readonly double PI = 3.14159;
    public static double DaireAlanHesapla(double yaricap) => PI * yaricap * yaricap;
}

// Kullanımı:
double alan = Matematik.DaireAlanHesapla(5);
Console.WriteLine($"PI Değeri: {Matematik.PI}");

// Örnek 2: Static sınıf (Tüm üyeleri static olmalı)
public static class Donusturucu
{
    public static double SantimetreyiInceCevir(double cm) => cm / 2.54;
}

// Kullanımı:
double inc = Donusturucu.SantimetreyiInceCevir(10);

// Örnek 3: Static constructor
public class Veritabani
{
    private static readonly string BaglantiCumlesi;
    static Veritabani() // Sadece bir kez, ilk kullanımda çalışır
    {
        Console.WriteLine("Static constructor çalıştı.");
        BaglantiCumlesi = "Server=myServer;Database=myDataBase;";
    }
    public static void Baglan() => Console.WriteLine($"{BaglantiCumlesi} ile bağlanıldı.");
}

Veritabani.Baglan(); // İlk çağrıda static constructor çalışır
Veritabani.Baglan(); // İkinci çağrıda çalışmaz

// Örnek 4: Normal sınıfta static sayaç
public class Islem
{
    public static int IslemSayisi { get; private set; } = 0;
    
    public Islem()
    {
        IslemSayisi++; // Her nesne oluşturulduğunda artar
    }
}
var islem1 = new Islem();
var islem2 = new Islem();
Console.WriteLine($"Toplam işlem sayısı: {Islem.IslemSayisi}"); // Çıktı: 2

// Örnek 5: Extension Metot (static sınıf içinde)
public static class StringExtensions
{
    public static string TersCevir(this string str)
    {
        char[] chars = str.ToCharArray();
        Array.Reverse(chars);
        return new string(chars);
    }
}
string metin = "Merhaba";
Console.WriteLine(metin.TersCevir()); // Çıktı: abahreM

// Örnek 6: Static metot ile kullanıcı girdisini işleme
Console.Write("İnç'e çevrilecek santimetreyi girin: ");
double cm = Convert.ToDouble(Console.ReadLine());
double inchResult = Donusturucu.SantimetreyiInceCevir(cm);
Console.WriteLine($"{cm} cm = {inchResult:F2} inç");

Sıra Sende! 🧑‍🤝‍🧑

  • `Logger` adında, `Log(string mesaj)` `static` metoduna sahip bir `static` sınıf oluşturun.
  • `YardimciFonksiyonlar` adında bir sınıf oluşturun ve içinde `static string BuyukHarfeCevir(string metin)` metodu yazın.
  • `Uygulama` adında bir sınıf oluşturun ve içinde uygulamanın versiyonunu tutan `public static string Versiyon` adında bir özellik tanımlayın.
  • `Kullanici` sınıfına, kaç tane kullanıcı nesnesi oluşturulduğunu sayan `static int KullaniciSayisi` adında bir özellik ekleyin.
  • `int` tipine, sayının karesini alan `KaresiniAl()` adında bir extension metot yazın.
  • Kullanıcıdan bir metin alın ve `YardimciFonksiyonlar` sınıfındaki `static` metodu kullanarak büyük harfe çevirip yazdırın.

Çözümler ✅

// 1. Logger static sınıfı
public static class Logger
{
    public static void Log(string mesaj) => Console.WriteLine($"LOG: {mesaj}");
}
Logger.Log("Uygulama başlatıldı.");

// 2. Yardımcı Fonksiyonlar
public class YardimciFonksiyonlar
{
    public static string BuyukHarfeCevir(string metin) => metin.ToUpper();
}
Console.WriteLine(YardimciFonksiyonlar.BuyukHarfeCevir("csharp"));

// 3. Static özellik
public class Uygulama
{
    public static string Versiyon { get; set; } = "1.0.0";
}
Console.WriteLine($"Uygulama Versiyonu: {Uygulama.Versiyon}");

// 4. Static sayaç
public class Kullanici
{
    public static int KullaniciSayisi { get; private set; }
    public Kullanici() { KullaniciSayisi++; }
}
var k1 = new Kullanici(); var k2 = new Kullanici();
Console.WriteLine($"Toplam kullanıcı: {Kullanici.KullaniciSayisi}");

// 5. Extension metot
public static class IntExtensions
{
    public static int KaresiniAl(this int sayi) => sayi * sayi;
}
int bes = 5;
Console.WriteLine($"5'in karesi: {bes.KaresiniAl()}");

// 6. Static metot ile kullanıcı girdisini işleme
Console.Write("Bir metin girin: ");
string metin = Console.ReadLine();
Console.WriteLine(YardimciFonksiyonlar.BuyukHarfeCevir(metin));

Mühürlü (Sealed) ve Parçalı (Partial) Sınıflar 🧱🧩

`sealed` ve `partial` anahtar kelimeleri sınıflara özel davranışlar kazandırır.

Mühürlü Sınıflar (Sealed)

Bir sınıf `sealed` olarak işaretlenirse, o sınıftan başka bir sınıf miras alamaz. Bu, sınıfın davranışının değiştirilmesini engellemek için kullanılır.

// Örnek 1: Sealed sınıf
public sealed class Ayarlar
{
    // ...
}
// public class GelismisAyarlar : Ayarlar {} // HATA! 'Ayarlar' sealed olduğu için miras alınamaz.

// Örnek 2: Sealed metot
public class Temel
{
    public virtual void Metot1() {}
}
public class Turemis : Temel
{
    public sealed override void Metot1() {} // Bu metot daha fazla override edilemez.
}
public class IleriTuremis : Turemis
{
    // public override void Metot1() {} // HATA! Metot1 sealed.
}

Parçalı Sınıflar (Partial)

Bir sınıfın tanımını birden fazla dosyaya bölmek için `partial` anahtar kelimesi kullanılır. Derleyici bu parçaları tek bir sınıf olarak birleştirir. Özellikle otomatik kod üreten araçlar (örn: WinForms, EF) tarafından sıkça kullanılır.

// Dosya1: Personel_Properties.cs
public partial class Personel
{
    public string Ad { get; set; }
    public int Yas { get; set; }
}

// Dosya2: Personel_Methods.cs
public partial class Personel
{
    public void BilgiYazdir()
    {
        Console.WriteLine($"{Ad}, {Yas} yaşında.");
    }
}
// Kullanım: Derleyici bu iki dosyayı birleştirir.
var p = new Personel { Ad = "Can", Yas = 28 };
p.BilgiYazdir();

// Örnek 5: Parçalı metot (Partial Method)
// Bir parçada tanımı, diğer parçada implementasyonu yapılır.
// Dosya1.cs
public partial class Rapor {
    partial void Logla(string mesaj); // Tanım

    public void Olustur() {
        // ...
        Logla("Rapor oluşturuldu."); // Çağrı
    }
}
// Dosya2.cs
public partial class Rapor {
    partial void Logla(string mesaj) { // Implementasyon
        Console.WriteLine($"LOG: {mesaj}");
    }
}

Sıra Sende! 🧱🧩

  • `String` sınıfının neden `sealed` olduğunu bir cümleyle açıklayın.
  • `Guvenlik` adında `sealed` bir sınıf oluşturun.
  • `Araba` adında bir `partial` sınıf oluşturun. Bir dosyada özelliklerini (`Marka`, `Model`), diğer dosyada `Calistir()` metodunu tanımlayın.
  • Oluşturduğunuz `Araba` sınıfından bir nesne yaratıp `Calistir()` metodunu çağırın.
  • Neden `partial` sınıflara ihtiyaç duyulduğunu açıklayın.
  • Kullanıcıdan `Araba`'nın markasını ve modelini alarak `partial` sınıfın farklı dosyalardaki özelliklerini doldurun ve metodu çağırın.

Çözümler ✅


// 1. String sınıfı, temel ve değiştirilemez bir yapı olduğu için
// davranışının tutarlılığını korumak amacıyla mühürlenmiştir.

// 2. Sealed Guvenlik sınıfı
public sealed class Guvenlik {}

// 3. Partial Araba sınıfı
// Araba_Props.cs
public partial class Araba {
    public string Marka { get; set; }
    public string Model { get; set; }
}
// Araba_Methods.cs
public partial class Araba {
    public void Calistir() => Console.WriteLine($"{Marka} çalıştırıldı.");
}

// 4. Araba nesnesi kullanımı
var araba = new Araba { Marka = "Ford", Model = "Focus" };
araba.Calistir();

// 5. Partial sınıflar, büyük sınıfları yönetilebilir parçalara ayırmak ve 
// otomatik üretilen kod ile manuel yazılan kodu ayırmak için kullanılır.

// 6. Kullanıcıdan veri alarak partial sınıf kullanma
var kullaniciAraba = new Araba();
Console.Write("Araba Markası: ");
kullaniciAraba.Marka = Console.ReadLine();
Console.Write("Araba Modeli: ");
kullaniciAraba.Model = Console.ReadLine();
kullaniciAraba.Calistir();

İsimsiz Tipler (Anonymous Types) 🕵️

İsimsiz tipler, geçici olarak birkaç özelliği bir arada gruplamak için kullanılır. `new { ... }` sözdizimi ile oluşturulurlar ve özellikleri sadece okunabilirdir (read-only).

// Örnek 1: Temel isimsiz tip
var kullanici = new { Ad = "Ali", Yas = 30 };
Console.WriteLine($"Ad: {kullanici.Ad}, Yaş: {kullanici.Yas}");

// Örnek 2: LINQ sorgularında kullanımı
var urunler = new[] {
    new { Ad = "Laptop", Fiyat = 25000 },
    new { Ad = "Mouse", Fiyat = 500 }
};

var pahaliUrunler = urunler.Where(u => u.Fiyat > 1000);
foreach (var urun in pahaliUrunler)
{
    Console.WriteLine(urun.Ad);
}

// Örnek 3: İç içe isimsiz tipler
var siparis = new
{
    Id = 101,
    Musteri = new { Ad = "Veli", Sehir = "İzmir" },
    Tutar = 150.75
};
Console.WriteLine($"Müşteri Adı: {siparis.Musteri.Ad}");

// Örnek 4: Metottan isimsiz tip döndürme (genellikle tavsiye edilmez)
object GetKullanici()
{
    return new { Ad = "Zeynep", AktifMi = true };
}
dynamic user = GetKullanici();
Console.WriteLine(user.Ad);

// Örnek 5: İsimsiz tiplerin eşitliği
var p1 = new { X = 10, Y = 20 };
var p2 = new { X = 10, Y = 20 };
// Özelliklerin isimleri, tipleri ve değerleri aynı olduğu için true döner.
Console.WriteLine($"p1 ve p2 eşit mi? {p1.Equals(p2)}");

Sıra Sende! 🕵️‍♀️

  • `Kitap` bilgilerini (`Baslik`, `Yazar`) tutan bir isimsiz tip oluşturun.
  • `Id` ve `Ad` özelliklerine sahip birkaç isimsiz tipten oluşan bir dizi oluşturun.
  • Bu dizideki Id'si 2 olan elemanın adını yazdırın (LINQ kullanarak).
  • `Koordinat` bilgilerini (`X`, `Y`, `Z`) tutan bir isimsiz tip oluşturun.
  • İsimsiz tiplerin en yaygın kullanım alanının neresi olduğunu bir cümleyle açıklayın.
  • Kullanıcıdan bir ürün adı ve fiyatı alın ve bu bilgileri bir isimsiz tipte saklayarak ekrana yazdırın.

Çözümler ✅

var kitap = new { Baslik = "Sefiller", Yazar = "Victor Hugo" };

// 2. İsimsiz tip dizisi
var kategoriler = new[] {

    new { Id = 2, Ad = "Kitap" },
    new { Id = 3, Ad = "Giyim" }

// 3. LINQ ile eleman bulma
var kategori = kategoriler.FirstOrDefault(k => k.Id == 2);
if (kategori != null) {
    Console.WriteLine(kategori.Ad);
}

// 4. Koordinat isimsiz tipi
var koordinat = new { X = 10.5, Y = 20.2, Z = 5.0 };

// 5. Kullanım alanı: İsimsiz tipler en çok, LINQ sorgularından
// dönen verileri geçici olarak şekillendirmek için kullanılır.

// 6. Kullanıcıdan alınan veriyle isimsiz tip oluşturma
Console.Write("Ürün adı: ");
string ad = Console.ReadLine();

double fiyat = Convert.ToDouble(Console.ReadLine());

Console.WriteLine($"Eklenen Ürün: {urun.Ad}, Fiyatı: {urun.Fiyat} TL");

Hata Yönetimi (Try-Catch) 🛡️

`try-catch` blokları, programınızın çökmesine neden olabilecek hataları (istisnaları/exceptions) yakalamak ve yönetmek için kullanılır. `finally` bloğu ise hata olsa da olmasa da çalışır.

// Örnek 1: Format hatası
try
{
    int sayi = int.Parse("abc"); // Bu satır hata fırlatacak
}
catch (FormatException ex) // Hata burada yakalanır
{
    Console.WriteLine($"Hata: Geçersiz sayı formatı. {ex.Message}");
}
finally
{
    Console.WriteLine("İşlem tamamlandı.");
}
// Örnek 2: Sıfıra bölme hatası
int a = 10, b = 0;
try
{
    Console.WriteLine(a / b);
}
catch (DivideByZeroException)
{
    Console.WriteLine("Hata: Sıfıra bölme hatası.");
}

// Örnek 3: Birden fazla catch bloğu
try
{
    int[] arr = { 1, 2 };
    Console.WriteLine(arr[5]); // IndexOutOfRangeException
}
catch (IndexOutOfRangeException)
{
    Console.WriteLine("Hata: Dizi sınırları dışında bir indekse erişim.");
}
catch (Exception ex) // Diğer tüm hatalar için
{
    Console.WriteLine($"Genel bir hata oluştu: {ex.GetType()}");
}
// Örnek 4: Kendi istisnamızı fırlatma (throw)
void DosyaOku()
{
    try
    {
        throw new FileNotFoundException("Dosya bulunamadı!");
    }
    catch (FileNotFoundException ex)
    {
        Console.WriteLine($"Hata yakalandı: {ex.Message}");
    }
}
DosyaOku();

// Örnek 5: Hata filtresi (Exception Filter - C# 6.0+)
try
{
    throw new HttpRequestException("Timeout", null, System.Net.HttpStatusCode.RequestTimeout);
}
catch (HttpRequestException ex) when (ex.StatusCode == System.Net.HttpStatusCode.NotFound)
{
    Console.WriteLine("Sayfa bulunamadı.");
}
catch (HttpRequestException ex) when (ex.StatusCode == System.Net.HttpStatusCode.RequestTimeout)
{
    Console.WriteLine("İstek zaman aşımına uğradı.");
}

// Örnek 6: Kullanıcıdan alınan yaş girdisini try-catch ile kontrol etme
Console.Write("Lütfen yaşınızı girin: ");
try
{
    int yas = int.Parse(Console.ReadLine());
    Console.WriteLine($"Yaşınız: {yas}");
}
catch (FormatException)
{
    Console.WriteLine("Hata: Lütfen geçerli bir sayı girin.");
}

Sıra Sende! ⚠️

  • Kullanıcıdan bir metin girmesini isteyin. Eğer girilen metin bir sayıya çevrilemiyorsa `FormatException`'ı yakalayıp "Geçersiz giriş" mesajı verin.
  • `bolme` metodunda ikinci parametre 0 ise `DivideByZeroException` hatasını yakalayın.
  • Bir `string` listesi oluşturun ve 5. indeksteki elemana erişmeye çalışın. `ArgumentOutOfRangeException` hatasını yakalayın.
  • Bir `try` bloğu içinde `throw new InvalidOperationException("Geçersiz işlem");` ifadesiyle bir hata fırlatın ve `catch` bloğunda yakalayın.
  • Bir `try-catch` bloğu oluşturun ve hem `catch` hem de `finally` bloklarının içine birer `Console.WriteLine` ifadesi ekleyerek her iki bloğun da çalıştığını gösterin.
  • Kullanıcıdan doğum yılını isteyin ve yaşını hesaplayın. Girilen değer sayı değilse `try-catch` ile hatayı yönetin.

Hata Ayıklama (Debugging) 🐞

Hata ayıklama, kodunuzdaki hataları bulma ve düzeltme sürecidir. Visual Studio gibi IDE'ler bu süreç için güçlü araçlar sunar.

Temel Teknikler

  • Breakpoint (Kesme Noktası): Kodun belirli bir satırda durmasını sağlar. Değişkenlerin o anki değerlerini inceleyebilirsiniz. (Genellikle F9 ile konulur).
  • Step Over (F10): Kodu satır satır çalıştırır. Metot çağrılarına girmez, metodun sonucunu bekler.
  • Step Into (F11): Kodu satır satır çalıştırır. Bir metot çağrısına denk gelirse, o metodun içine girer.
  • Step Out (Shift+F11): İçinde bulunduğunuz metottan çıkıp, çağrıldığı yere geri döner.
  • Watch/Locals/Autos Pencereleri: Değişkenlerin değerlerini anlık olarak izlemenizi sağlar.
// Örnek 1: Console.WriteLine ile hata ayıklama
int a = 5;
int b = 10;
Console.WriteLine($"Döngüye girmeden önce a: {a}, b: {b}");
for(int i = 0; i < 3; i++)
{
    a += i;
    b *= i;
    Console.WriteLine($"Döngü {i}. adım: a={a}, b={b}");
}
Console.WriteLine($"Sonuç: {a+b}");

  • Step Over (F10) ve Step Into (F11) arasındaki fark nedir?
  • Bir `for` döngüsü içinde, döngü değişkeninin değerini her adımda `Console.WriteLine` ile yazdırın.
  • Bir metot yazın, metot içinde bir değişkenin değerini `Debug.WriteLine` ile yazdırın.
  • Aşağıdaki kodda mantık hatasını bulun ve düzeltin: `string metin = "Merhaba"; if (metin.Length > 5) { Console.WriteLine("Uzun metin"); }`
