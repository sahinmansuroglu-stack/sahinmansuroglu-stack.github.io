Görev Çözümleri

Aşağıda readme.html dosyasındaki 5 temel görevin bağımsız çözümleri yer almaktadır. Bu kodlar functional-app için birer ön çalışma niteliğindedir.

← Görevlere Dön

1. Çözüm: FormInput Kullanımı

Kullanıcıdan başlık bilgisini almak için:

<script>
    import FormInput from "$lib/components/FormInput.svelte";
    
    // Veriyi tutacak değişken
    let baslik = $state("");
</script>

<!-- Kullanımı -->
<FormInput 
    label="Başlık" 
    placeholder="Örn: Alışveriş" 
    bind:value={baslik} 
/>

<p>Yazılan Değer: {baslik}</p>
Bu kodu boş bir Svelte sayfasına yapıştırarak test edebilirsiniz.

2. Çözüm: Card Kullanımı

Veriyi kart içinde göstermek ve butona tıklama olayını yakalamak için:

<script>
    import Card from "$lib/components/Card.svelte";
    import cardImage from "$lib/images/card-image.png";

    function sil() {
        alert("Silme fonksiyonu çağrıldı!");
    }
</script>

<div style="width: 300px;">
    <Card
        title="Örnek Başlık"
        imageUrl={cardImage}
        buttonText="Sil"
        onclick={sil}
    >
        Bu alan kartın slot kısmıdır. Açıklama buraya gelir.
    </Card>
</div>

3. Çözüm: Modal Kullanımı

Silme onayı için açılır pencere (Modal) kullanımı:

<script>
    import Modal from "$lib/components/Modal.svelte";

    let modalAcik = $state(false);

    function onayla() {
        alert("Silme onaylandı!");
        modalAcik = false;
    }
</script>

<button class="btn btn-primary" onclick={() => modalAcik = true}>
    Modalı Aç
</button>

<Modal
    title="Silme Onayı"
    bind:open={modalAcik}
    onsave={onayla}
>
    <p>Bu öğeyi silmek istediğinize emin misiniz?</p>
</Modal>

4. Çözüm: Alert Kullanımı

Hata mesajını göstermek için:

<script>
    import Alert from "$lib/components/Alert.svelte";
    
    let hata = $state("Bir sorun oluştu!");
</script>

{#if hata}
    <Alert 
        type="danger" 
        message={hata} 
        onclose={() => (hata = null)} 
    />
{:else}
    <button class="btn btn-warning" onclick={() => hata = "Yeni hata!"}>
        Hata Oluştur
    </button>
{/if}

5. Çözüm: Spinner Kullanımı

Yükleniyor durumuna göre buton yerine spinner göstermek için:

<script>
    import Spinner from "$lib/components/Spinner.svelte";
    
    let yukleniyor = $state(false);

    function islemYap() {
        yukleniyor = true;
        setTimeout(() => yukleniyor = false, 2000);
    }
</script>

{#if yukleniyor}
    <div class="d-flex align-items-center">
        <Spinner type="border" color="primary" size="sm" />
        <span class="ms-2">İşleniyor...</span>
    </div>
{:else}
    <button class="btn btn-success" onclick={islemYap}>
        Kaydet
    </button>
{/if}

6. Çözüm: Navbar Kullanımı

Sayfa navigasyonu için:

<script>

    ];

<Navbar 
    brand="Benim Uygulamam" 
    links={navigasyonLinkleri} 
/>

7. Çözüm: Accordion Kullanımı

Sıkça sorulan sorular veya katlanabilir içerik için:

<script>
    import Accordion from "$lib/components/Accordion.svelte";
    
    const sss = [
        { title: "Svelte nedir?", content: "Svelte bir UI derleyicisidir." },
        { title: "Neden Svelte?", content: "Hızlı, reaktif ve az kod." },
        { title: "Runes nedir?", content: "Svelte 5 ile gelen yeni durum yönetim sistemi." }
    ];
</script>

<Accordion items={sss} />

8. Çözüm: Basit Fade Animasyonu

Bileşenlerin yumuşak bir şekilde görünmesi veya kaybolması için:

<script>
    import { fade } from "svelte/transition";
    let goster = $state(true);
</script>

{#if goster}
    <div transition:fade>
        Bu kutu animasyonla gelir/gider.
    </div>
{/if}

<button onclick={() => goster = !goster}>Değiştir</button>

9. Çözüm: Özelleştirilmiş Fade

Animasyonun süresini, gecikmesini veya easing fonksiyonunu değiştirmek için:

<script>
    import { fade } from "svelte/transition";
</script>

<div transition:fade={{ duration: 2000, delay: 500 }}>
    Bu yazı 0.5 saniye bekler, sonra 2 saniyede yavaşça belirir.
</div>
