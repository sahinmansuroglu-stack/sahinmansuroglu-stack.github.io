Modern Dapper Rehberi

ORM Dünyasına ve Asenkron Programlamaya Giriş

📦 1. ORM ve Dapper Nedir?

ORM Nedir?

Object-Relational Mapping (Nesne-İlişkisel Eşleme). Veritabanındaki tabloları C# içindeki sınıflara (class) otomatik olarak bağlayan bir köprüdür. SQL sorgularıyla uğraşırken verileri elle tek tek kutulara doldurmaktan bizi kurtarır.

Dapper Nedir?

Stack Overflow ekibi tarafından geliştirilen "Micro-ORM"dur. Çok hızlıdır, hafiftir ve SQL yazma özgürlüğünü bozmadan verileri otomatik olarak nesnelere dönüştürür. Yazılımcılar arasında "Hız canavarı" olarak bilinir.

⏳ 2. Asenkron Programlama: Task, Async, Await

Modern programlamada asenkron yapı çok önemlidir. Programımızın "donmamasını" (kilitlenmemesini) sağlar.

Mutfak Metaforu:
Bir mutfak düşünün. Makarnayı haşlamak için suyu ateşe koydunuz. Suyun kaynaması (veritabanından cevap gelmesi) 10 dakika sürecek.

Senkron (Eski Yöntem): Su kaynayana kadar ocağın başında put gibi beklersiniz. Başka hiçbir şey yapamazsınız. (Program donar!)
Asenkron (Modern Yöntem): Suyu koyarsınız, o kaynarken siz salatayı yapmaya (arayüzü canlı tutmaya, kullanıcıya cevap vermeye) devam edersiniz.

Task (Görev)

Yapılacak, zaman alacak işin kendisidir. "Bu bir Görev'dir (Task), hemen bitmeyebilir, haberin olsun" demektir. Örneğin veritabanından binlerce öğrenci kaydını çekmek bir Task'tır.

async (Asenkron Metot)

Bir metodun asenkron çalışacağını ve içinde await kullanılabileceğini bilgisayara bildiren etikettir.

await (Bekle)

O görev bitene kadar sadece o satırda "bekle", ama programın geri kalanını kilitleme! Görev bittiğinde içindeki sonucu al ve bir sonraki satırdan kodları çalıştırmaya devam et demektir.

💻 3. Dapper ile Tam Kapsamlı (Statik) Kodlar

Şimdi bir öğrenci veritabanı üzerinden işlemleri görelim. Kullanıcıdan TextBox ile veri almak yerine, daha iyi anlaman için verileri kodun içinde (statik olarak) verdik.

Veritabanı Sınıfı (Entity Class)

Veritabanımızdaki tablo ile birebir eşleşen sınıfımız. Özellik (Property) isimleri veritabanı sütunlarıyla aynı olmalıdır.

public class Ogrenci 
{
    public int OgrenciID { get; set; }
    public string Ad { get; set; }
    public string Soyad { get; set; }
    public string Sinif { get; set; }
    public int Puan { get; set; }
}

Bağlantı (Connection) Ayarları

Veritabanının nerede olduğunu ve şifresini belirten ConnectionString ile bir kapı oluşturuyoruz.

using Dapper;
using MySql.Data.MySqlClient;
using System.Data;
using System.Threading.Tasks;
using System.Linq;

// ... sınıfın içi ...

string connectionString = "host=localhost;port=3306;user id=root;password=mtal123;database=okuldb;";

// Bağlantıyı tek bir yerden yöneten özellik
private IDbConnection Connection => new MySqlConnection(connectionString);

🔍 A. Listeleme İşlemi (Read)

Veritabanından tüm kayıtları çekip C# listesine çevirme işlemi.

public async Task Listele()
{
    // using: İşimiz bitince veritabanı kapısını otomatik kapatır
    using var db = Connection;
    string sql = "SELECT * FROM ogrenciler";

    // QueryAsync: Veritabanından veri (liste) ÇEKMEK için kullanılır
    var liste = (await db.QueryAsync<Ogrenci>(sql)).ToList();
   
    dataGridView1.DataSource = liste;
}

QueryAsync<Ogrenci> : Dapper'ın en güçlü metodudur! SQL'den gelen her bir satırı otomatik olarak Ogrenci sınıfına dönüştürür. Sonunda ToList() diyerek bunları bir Listeye çeviririz. Çünkü veriyi veritabanından getirmek (Query) istiyoruz.

➕ B. Ekleme İşlemi (Create)

Yeni bir öğrenciyi veritabanına kaydetme işlemi.

public async Task YeniOgrenciEkle()
{
    using var db = Connection;
    
    // Statik olarak verileri kendimiz dolduruyoruz
    var yeniOgrenci = new Ogrenci {
        Ad = "Ali",
        Soyad = "Veli",
        Sinif = "10-A",
        Puan = 85
    };

    // @Ad, @Soyad gibi ifadeler Dapper'ın güvenli parametreleridir
    string sql = "INSERT INTO ogrenciler (Ad, Soyad, Sinif, Puan) VALUES (@Ad, @Soyad, @Sinif, @Puan)";
    
    // ExecuteAsync: Veritabanında DEĞİŞİKLİK YAPMAK (Ekle, Sil, Güncelle) için kullanılır
    await db.ExecuteAsync(sql, yeniOgrenci);
    
    await Listele(); // İşlem bitince listeyi yenile
}

ExecuteAsync : Veritabanında bir işlem "icra etmek" (çalıştırmak) içindir. Ekleme (Insert) işleminde geriye bir liste dönmesini beklemeyiz. Sadece eylemin gerçekleşmesini isteriz. Bu yüzden QueryAsync yerine ExecuteAsync kullanılır. İkinci parametre olan yeniOgrenci içindeki verileri, Dapper otomatik olarak SQL'deki @Ad, @Soyad yerlerine yerleştirir.

✏️ C. Güncelleme İşlemi (Update)

Var olan bir kaydın bilgilerini ID'sine göre değiştirme işlemi.

public async Task OgrenciGuncelle()
{
    using var db = Connection;
    
    // ID'si 1 olan öğrencinin puanını 95 olarak güncelliyoruz
    var guncelOgrenci = new Ogrenci {
        OgrenciID = 1, // Hangi öğrenci?
        Ad = "Ali",
        Soyad = "Veli",
        Sinif = "10-A",
        Puan = 95     // Puan değişti
    };

    string sql = "UPDATE ogrenciler SET Ad=@Ad, Soyad=@Soyad, Sinif=@Sinif, Puan=@Puan WHERE OgrenciID=@OgrenciID";
    
    await db.ExecuteAsync(sql, guncelOgrenci);
    
    await Listele();
}

Yine ExecuteAsync kullandık! Çünkü veritabanında bir güncelleme eylemi yapıyoruz. Dapper, OgrenciID=@OgrenciID kısmını bizim nesnemizdeki 1 değeri ile eşleştirip doğru kişiyi günceller.

🗑️ D. Silme İşlemi (Delete)

Sadece ID bilgisini göndererek veritabanından kaydı tamamen kaldırma.

public async Task OgrenciSil()
{
    using var db = Connection;
    string sql = "DELETE FROM ogrenciler WHERE OgrenciID = @id";
    
    // Geçici (Anonymous) nesne oluşturarak id'yi Dapper'a gönderiyoruz
    await db.ExecuteAsync(sql, new { id = 5 }); // ID'si 5 olanı sil
    
    await Listele();
}

new { id = 5 } : Tüm Ogrenci sınıfını göndermek yerine, sadece silmek için gereken @id bilgisini geçici (isimsiz/anonymous) bir nesne olarak Dapper'a gönderdik. Dapper bunu anında SQL'in içine gömer ve işlemi gerçekleştirir.

🔎 E. Arama İşlemi (Search / Filter)

Belirli bir kelimeyi (örn: "Ahmet") veritabanında Ad veya Soyad içinde arama işlemi.

public async Task OgrenciAra()
{
    using var db = Connection;
    
    // Statik olarak 'Ahmet' kelimesini arıyoruz
    // % işaretleri SQL'de "öncesi ve sonrası önemli değil, içinde geçsin" demektir
    string aranan = "%Ahmet%"; 
    
    string sql = "SELECT * FROM ogrenciler WHERE Ad LIKE @metin OR Soyad LIKE @metin";
    
    // Veri (Liste) geri DÖNECEĞİ için QueryAsync kullanıyoruz!
    var sonuclar = (await db.QueryAsync<Ogrenci>(sql, new { metin = aranan })).ToList();
    
    dataGridView1.DataSource = sonuclar;
}

Burada tekrar QueryAsync komutuna döndük! Çünkü arama sonucunda bize eşleşen öğrencilerin listesi (bir sonuç kümesi) geliyor. Arama kelimesini new { metin = aranan } ile güvenli bir şekilde SQL'e gönderiyoruz.

📌 Özet Hatırlatma

  • ✅ ExecuteAsync: Veritabanında bir eylem/değişiklik (Ekle, Sil, Güncelle) yapacağımızda kullanılır. Geriye veri (liste) dönmez.
  • ✅ QueryAsync: Veritabanından veri çekmek (Listeleme, Arama) istediğimizde kullanılır. Bize Dapper sayesinde doğrudan nesne listesi olarak geri döner.
  • ✅ Using Bloğu: İşlem bittiğinde arka planda veritabanı kapısının (connection) otomatik kapanmasını sağlayıp sistemi rahatlatır.
  • ✅ await: Asenkron işlemin (örneğin ExecuteAsync'nin) veritabanında çalışmasını tamamlayana kadar beklemesini, ardından sıradaki koda (örneğin Listele metoduna) geçmesini sağlar.
