💡 Neden İlişkisel Veritabanı?

Veri Tasarrufu

Filmin adını her izleme kaydına tekrar tekrar yazmak yerine sadece "M1" yazıyoruz. Binlerce kayıt olduğunu düşünürseniz bu devasa bir yer tasarrufu sağlar.

Merkezi Yönetim

Bir filmin adı değişirse (örneğin yazım hatası düzeltilirse), sadece tek bir yerden (Filmler Tablosu) değiştirmek yeterli olur.

Güçlü Analiz

"Bilim kurgu seven kullanıcılar kim?" gibi soruları tabloları birbirine bağlayarak (JOIN) anında cevaplayabiliriz.

💻 SQL Sorguları (Tablo Oluşturma)

Aşağıdaki sorguları MySQL veya SQL Server üzerinde çalıştırarak bu yapıyı kendi bilgisayarınızda kurabilirsiniz.

SQL SCRIPT
-- 1. Kullanıcılar Tablosunu Oluştur
CREATE TABLE Users (
    KullaniciID INT PRIMARY KEY AUTO_INCREMENT,
    KullaniciAdi VARCHAR(50) NOT NULL,
    Sifre VARCHAR(100) NOT NULL,
    PaketTuru VARCHAR(20)
);

-- 2. Filmler Tablosunu Oluştur
CREATE TABLE Movies (
    FilmID VARCHAR(10) PRIMARY KEY,
    FilmAdi VARCHAR(100) NOT NULL,
    Tür VARCHAR(50),
    Sure INT
);

-- 3. İzleme Geçmişi Tablosunu Oluştur (İlişkili Tablo)
CREATE TABLE Watchlist (
    KayitID INT PRIMARY KEY AUTO_INCREMENT,
    KullaniciID INT,
    FilmID VARCHAR(10),
    VerilenPuan INT CHECK (VerilenPuan <= 10),
    Durum VARCHAR(20),
    -- İlişkileri Kur (Foreign Keys)
    FOREIGN KEY (KullaniciID) REFERENCES Users(KullaniciID),
    FOREIGN KEY (FilmID) REFERENCES Movies(FilmID)
);

Örnek Veri Ekleme

INSERT INTO Users (KullaniciAdi, Sifre, PaketTuru) VALUES ('Mert_2024', '123456', 'Premium');
INSERT INTO Movies (FilmID, FilmAdi, Tür, Sure) VALUES ('M1', 'Inception', 'Bilim Kurgu', 148);
INSERT INTO Watchlist (KullaniciID, FilmID, VerilenPuan, Durum) VALUES (1, 'M1', 10, 'Tamamlandı');

🔗 Verileri Birleştirme (JOIN Sorguları)

Tabloları birbirine bağlayarak anlamlı raporlar oluşturabiliriz. İşte en çok kullanılan JOIN örnekleri:

1. Hangi Kullanıcı Hangi Filmi İzledi? (İki Tablo)

Sadece izleme geçmişini ve kullanıcı adlarını birleştiririz.

SELECT 
    Users.KullaniciAdi, 
    Watchlist.FilmID, 
    Watchlist.VerilenPuan 
FROM Watchlist
INNER JOIN Users ON Watchlist.KullaniciID = Users.KullaniciID;

2. Tam Detaylı Rapor (Üç Tabloyu Birleştirme)

Kullanıcı adı, filmin gerçek adı, puanı ve durumunu tek bir tabloda görürüz.

SELECT 
    U.KullaniciAdi, 
    M.FilmAdi, 
    W.VerilenPuan, 
    W.Durum
FROM Watchlist W
INNER JOIN Users U ON W.KullaniciID = U.KullaniciID
INNER JOIN Movies M ON W.FilmID = M.FilmID;

💡 JOIN Mantığı

JOIN işlemi, tablolar arasındaki ortak sütunları (Örn: KullaniciID) kullanarak sanki tek bir büyük tabloyormuş gibi verileri yan yana getirir.
