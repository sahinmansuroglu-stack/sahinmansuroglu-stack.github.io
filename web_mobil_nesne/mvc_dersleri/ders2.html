MVC Mimarisi Detaylı

Model, View ve Controller Derinlemesine

MVC Akış Şeması

Bir kullanıcı isteği geldiğinde MVC'de şu adımlar gerçekleşir:

  1. Kullanıcı tarayıcıdan bir URL'e istek gönderir
  2. Router isteği uygun Controller'a yönlendirir
  3. Controller gerekli verileri Model'den alır
  4. Controller, verileri View'e gönderir
  5. View, HTML oluşturup kullanıcıya döner
💡 Separation of Concerns: MVC'nin temel prensibi, her katmanın kendi işine odaklanmasıdır. Bu sayede kod daha okunabilir ve bakımı kolaydır.

Controller Detaylı

Controller, uygulamanın "beyni"dir. İstekleri karşılar ve ne yapılacağına karar verir.

Örnek: ProductController.cs
using Microsoft.AspNetCore.Mvc;

public class ProductController : Controller
{
    // GET: /Product/Index
    public IActionResult Index()
    {
        // Tüm ürünleri listele
        return View();
    }

    // GET: /Product/Details/5
    public IActionResult Details(int id)
    {
        // id=5 olan ürünün detaylarını göster
        ViewBag.ProductId = id;
        return View();
    }

    // GET: /Product/Create
    public IActionResult Create()
    {
        // Yeni ürün ekleme formu
        return View();
    }
}

Action Metodları ve Dönüş Tipleri

Dönüş Tipi Açıklama Kullanım
View() Razor View döner HTML sayfa
RedirectToAction() Başka action'a yönlendir Form sonrası
Json() JSON veri döner API çağrıları
Content() Düz metin döner Basit yanıtlar
NotFound() 404 hatası döner Kayıt bulunamadı

Model Detaylı

Model, uygulamanın veri yapısını temsil eder. Genellikle veritabanı tablolarına karşılık gelir.

Örnek: Product.cs (Model)
namespace MyApp.Models
{
    public class Product
    {
        public int Id { get; set; }
        public string Name { get; set; }
        public decimal Price { get; set; }
        public string Description { get; set; }
        public int Stock { get; set; }
        public bool IsActive { get; set; }
    }
}
Örnek: Student.cs (Model)
public class Student
{
    public int Id { get; set; }
    public string FirstName { get; set; }
    public string LastName { get; set; }
    public int Grade { get; set; }  // Sınıf
    public string Email { get; set; }
    
    // Hesaplanmış property
    public string FullName => $"{FirstName} {LastName}";
}

Controller'dan View'e Veri Gönderme

1. ViewBag Kullanımı

Controller
public IActionResult Index()
{
    ViewBag.Message = "Hoş Geldiniz!";
    ViewBag.CurrentDate = DateTime.Now;
    ViewBag.StudentCount = 35;
    return View();
}
View (Index.cshtml)
<h1>@ViewBag.Message</h1>
<p>Tarih: @ViewBag.CurrentDate</p>
<p>Öğrenci Sayısı: @ViewBag.StudentCount</p>

2. ViewData Kullanımı

Controller
public IActionResult Details()
{
    ViewData["Title"] = "Ürün Detayı";
    ViewData["Price"] = 99.99;
    return View();
}
View
<h1>@ViewData["Title"]</h1>
<p>Fiyat: @ViewData["Price"] TL</p>

3. Model Kullanımı (Önerilen)

Controller
public IActionResult Details(int id)
{
    var product = new Product
    {
        Id = id,
        Name = "Laptop",
        Price = 15000,
        Stock = 10
    };
    return View(product);
}
View (Details.cshtml)
@model Product

<h1>@Model.Name</h1>
<p>Fiyat: @Model.Price TL</p>
<p>Stok: @Model.Stock adet</p>

Tam Örnek: Öğrenci Listesi

Models/Student.cs
public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Grade { get; set; }
}
Controllers/StudentController.cs
public class StudentController : Controller
{
    public IActionResult Index()
    {
        var students = new List<Student>
        {
            new Student { Id = 1, Name = "Ali", Grade = 11 },
            new Student { Id = 2, Name = "Ayşe", Grade = 11 },

        };
        return View(students);
    }
}

🎯 Sıra Sende!

1 ViewBag ve Model arasındaki fark nedir? Hangisi tercih edilmeli?

ViewBag: Dinamik, tip güvenliği yok, küçük veriler için uygun

Model: Tip güvenli, IntelliSense desteği, büyük/karmaşık veriler için ideal

Öneri: Her zaman Model kullanın, tip hatalarını compile-time'da yakalar.

2 "Book" adında Id, Title, Author, Price özellikleri olan bir Model sınıfı yazın. 
public class Book
{
    public int Id { get; set; }
    public string Title { get; set; }
    public string Author { get; set; }
    public decimal Price { get; set; }
}
3 BookController'da 3 kitap içeren bir liste oluşturup View'e gönderen Index action'ı yazın. 
public class BookController : Controller
{
    public IActionResult Index()
    {
        var books = new List<Book>
        {
            new Book { Id = 1, Title = "C# Temelleri", Author = "Ahmet Yılmaz", Price = 75 },
            new Book { Id = 2, Title = "ASP.NET MVC", Author = "Fatma Kaya", Price = 95 },
            new Book { Id = 3, Title = "SQL Server", Author = "Can Demir", Price = 85 }
        };
        return View(books);
    }
}
