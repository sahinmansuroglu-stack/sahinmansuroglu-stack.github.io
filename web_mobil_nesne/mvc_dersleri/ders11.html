Dependency Injection ve Servisler

Temiz Mimari ve Bağımlılık Yönetimi

Dependency Injection Nedir?

Dependency Injection (DI), bir sınıfın bağımlılıklarını dışarıdan almasını sağlayan tasarım kalıbıdır.

💡 Neden DI Kullanırız?
• Kodun test edilebilirliği artar
• Bağımlılıklar gevşek bağlı olur (loose coupling)
• Kodun bakımı kolaylaşır
• Aynı arayüzü farklı implementasyonlarla değiştirebiliriz

Service Katmanı Oluşturma

Services/IStudentService.cs
public interface IStudentService
{
    Task<List<Student>> GetAllAsync();
    Task<Student> GetByIdAsync(int id);
    Task CreateAsync(Student student);
    Task UpdateAsync(Student student);
    Task DeleteAsync(int id);
    Task<List<Student>> SearchAsync(string searchTerm);
}
Services/StudentService.cs
public class StudentService : IStudentService
{
    private readonly SchoolDbContext _context;
    
    public StudentService(SchoolDbContext context)
    {
        _context = context;
    }
    
    public async Task<List<Student>> GetAllAsync()
    {
        return await _context.Students
            .Include(s => s.Classroom)
            .OrderBy(s => s.LastName)
            .ToListAsync();
    }
    
    public async Task<Student> GetByIdAsync(int id)
    {
        return await _context.Students
            .Include(s => s.Classroom)
            .FirstOrDefaultAsync(s => s.Id == id);
    }
    
    public async Task CreateAsync(Student student)
    {
        _context.Students.Add(student);
        await _context.SaveChangesAsync();
    }
    
    public async Task UpdateAsync(Student student)
    {
        _context.Students.Update(student);
        await _context.SaveChangesAsync();
    }
    
    public async Task DeleteAsync(int id)
    {
        var student = await _context.Students.FindAsync(id);
        if (student != null)
        {
            _context.Students.Remove(student);
            await _context.SaveChangesAsync();
        }
    }
    
    public async Task<List<Student>> SearchAsync(string searchTerm)
    {
        return await _context.Students
            .Where(s => s.FirstName.Contains(searchTerm) || 
                       s.LastName.Contains(searchTerm))
            .ToListAsync();
    }
}

Program.cs'de Kayıt

Program.cs
var builder = WebApplication.CreateBuilder(args);

// DbContext
builder.Services.AddDbContext<SchoolDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));

// Servis kayıtları
builder.Services.AddScoped<IStudentService, StudentService>();
builder.Services.AddScoped<IClassroomService, ClassroomService>();

// MVC
builder.Services.AddControllersWithViews();

var app = builder.Build();
⚠️ Servis Yaşam Döngüleri:
• AddScoped: Her HTTP isteği için yeni instance
• AddTransient: Her kullanımda yeni instance
• AddSingleton: Uygulama boyunca tek instance

Controller'da Kullanım

Controllers/StudentController.cs
public class StudentController : Controller
{
    private readonly IStudentService _studentService;
    private readonly IClassroomService _classroomService;
    
    // Constructor Injection
    public StudentController(
        IStudentService studentService,
        IClassroomService classroomService)
    {
        _studentService = studentService;
        _classroomService = classroomService;
    }
    
    public async Task<IActionResult> Index()
    {
        var students = await _studentService.GetAllAsync();
        return View(students);
    }
    
    public async Task<IActionResult> Details(int id)
    {
        var student = await _studentService.GetByIdAsync(id);
        if (student == null) return NotFound();
        return View(student);
    }
    
    [HttpPost]
    public async Task<IActionResult> Create(Student student)
    {
        if (ModelState.IsValid)
        {
            await _studentService.CreateAsync(student);
            TempData["Success"] = "Öğrenci eklendi!";
            return RedirectToAction(nameof(Index));
        }
        return View(student);
    }
}

2 AddScoped, AddTransient, AddSingleton arasındaki fark nedir?
  • Scoped: Her HTTP isteği için bir instance oluşturulur
  • Transient: Her enjeksiyon için yeni instance
  • Singleton: Uygulama boyunca tek instance
