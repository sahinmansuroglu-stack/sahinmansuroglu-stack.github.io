Form İşlemleri ve HTTP Metodları

GET, POST ve Form Binding

HTTP Metodları

Metod Kullanım Örnek
GET Veri okuma, sayfa görüntüleme Liste, detay sayfası
POST Veri gönderme, oluşturma Form gönderme, kayıt ekleme
PUT Veri güncelleme Kayıt düzenleme
DELETE Veri silme Kayıt silme
💡 Web formlarında sadece GET ve POST kullanılır. PUT/DELETE API'lerde kullanılır.

HttpGet ve HttpPost Attributes

Controllers/ProductController.cs
using Microsoft.AspNetCore.Mvc;

public class ProductController : Controller
{
    private readonly AppDbContext _context;
    
    public ProductController(AppDbContext context)
    {
        _context = context;
    }

    // GET: /Product/Create - Boş form göster
    [HttpGet]
    public IActionResult Create()
    {
        return View();
    }
    
    // POST: /Product/Create - Form verisi al ve kaydet
    [HttpPost]
    public IActionResult Create(Product product)
    {
        if (ModelState.IsValid)
        {
            // Veritabanına kaydet
            _context.Products.Add(product);
            _context.SaveChanges();
            return RedirectToAction("Index");
        }
        return View(product);
    }
}

Edit (Düzenleme) İşlemi

Controller - Edit
using Microsoft.AspNetCore.Mvc;

// GET: /Product/Edit/5
[HttpGet]
public IActionResult Edit(int id)
{
    var product = _context.Products.Find(id);
    if (product == null)
        return NotFound();
    
    return View(product);
}

// POST: /Product/Edit/5
[HttpPost]
public IActionResult Edit(int id, Product product)
{
    if (id != product.Id)
        return NotFound();
    
    if (ModelState.IsValid)
    {
        _context.Products.Update(product);
        _context.SaveChanges();
        
        TempData["Success"] = "Ürün güncellendi!";
        return RedirectToAction("Index");
    }
    return View(product);
}

Delete (Silme) İşlemi

Controller - Delete
// GET: /Product/Delete/5 - Onay sayfası
[HttpGet]
public IActionResult Delete(int id)
{
    var product = _context.Products.Find(id);
    if (product == null)
        return NotFound();
    
    return View(product);
}

// POST: /Product/Delete/5 - Gerçek silme
[HttpPost, ActionName("Delete")]
public IActionResult DeleteConfirmed(int id)
{
    var product = _context.Products.Find(id);
    if (product != null)
    {
        _context.Products.Remove(product);
        _context.SaveChanges();
        TempData["Success"] = "Ürün silindi!";
    }
    return RedirectToAction("Index");
}

🎯 Sıra Sende!

1 GET ve POST metodlarının farkını açıklayın.

GET:

  • Veri okumak için kullanılır
  • Parametreler URL'de görünür
  • Sayfa yenileme güvenlidir
  • Örnek: Listeleme, arama

POST:

  • Veri göndermek için kullanılır
  • Veriler body'de gizlidir
  • Form gönderimi için
  • Örnek: Kayıt, güncelleme
2 Öğrenci ekleme formu için Create GET ve POST metodlarını yazın. 
using Microsoft.AspNetCore.Mvc;

[HttpGet]
public IActionResult Create()
{
    return View();
}

[HttpPost]
public IActionResult Create(Student student)
{
    if (ModelState.IsValid)
    {
        _context.Students.Add(student);
        _context.SaveChanges();
        TempData["Success"] = "Öğrenci eklendi!";
        return RedirectToAction("Index");
    }
    return View(student);
}

4 Edit action'ında hidden field neden gereklidir?

Hidden field, düzenlenen kaydın Id değerini form ile birlikte gönderir.

Bu sayede POST metodunda hangi kaydın güncelleneceği bilinir.

<input type="hidden" asp-for="Id" />

Olmadan, Id değeri = 0 olur ve güncelleme başarısız olur.
